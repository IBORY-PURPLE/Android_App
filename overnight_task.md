## 반복 프로토콜 (= 피드백 루프)

각 깨어남에서:

1. PROGRESS.md와 DECISIONS_NEEDED.md를 읽어 현재 위치를 파악한다.
2. 남은 것 중 가장 작은 다음 한 걸음 하나를 고른다.
3. 필요하면 docs.expo.dev/versions/v57.0.0 에서 해당 API를 확인한다.
4. 최소 증분으로 구현한다.
5. 검증 게이트를 실행한다:
   - npx tsc --noEmit (타입 에러 0)
   - npx expo-doctor (통과)
   - npx expo export -p android (번들 무에러)
6. 실패하면 고쳐서 다시 검증한다(fix-forward). 3회 시도해도 안 되면
   DECISIONS_NEEDED.md에 원인을 적고 이 증분을 되돌린 뒤 다른 작업으로 넘어간다.
7. 통과하면 커밋하고 **origin master로 push한다**(`git push origin master`).
   (작고 명확한 메시지, 한 증분 = 한 커밋. push 실패는 치명 아님 — 기록하고 계속.)
8. PROGRESS.md를 갱신한다: 방금 한 것 / 검증 결과 / 다음 후보 / 지금까지 반복 횟수.
9. 검증 통과한 증분을 1개 이상 완료하고 PROGRESS.md를 최신으로 남긴 뒤 이 턴을 종료한다.
